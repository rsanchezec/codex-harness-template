# Architecture Principles

- Prefer managed services when possible.
- Prefer secure by default architectures.
- Prefer least privilege access.
- Prefer API-first integrations.
- Prefer event-driven architectures when appropriate.
- Prefer modular and loosely coupled components.
- Avoid unnecessary complexity.
- Avoid direct model access to critical systems.
- Keep business rules outside prompts.

---

# Agent Design Principles

When designing AI agents:

- Clearly define responsibilities.
- Minimize tool permissions.
- Use tool calling for transactional operations.
- Use RAG for knowledge retrieval.
- Separate knowledge from actions.
- Implement evaluation from day one.
- Design for observability and auditability.
