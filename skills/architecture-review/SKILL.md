# Architecture Review

Purpose

Evaluate architecture quality before implementation.

Ensure the solution is scalable, maintainable, secure, observable and deployable.

---

# Review Process

1. Understand Business Goals
2. Review System Boundaries
3. Review Components
4. Review Integrations
5. Review Security
6. Review Scalability
7. Review Observability
8. Review Deployment Strategy
9. Review Risks
10. Review Tradeoffs

---

# Review Areas

Evaluate:

- Maintainability
- Scalability
- Reliability
- Security
- Performance
- Cost
- Complexity
- Extensibility
- Operational Burden

---

# System Boundaries

Verify:

- Clear responsibilities
- Clear ownership
- Clear interfaces
- Clear dependencies

Flag:

- Tight coupling
- Circular dependencies
- Undefined ownership

---

# Integration Review

Review:

- API integrations
- Event-driven integrations
- Batch integrations
- External systems

Verify:

- Error handling
- Retry strategy
- Failure recovery

---

# Deployment Review

Always verify:

- Environment strategy
- Rollback strategy
- Monitoring strategy
- Disaster recovery

---

# Scalability Review

Review:

- Database scalability
- API scalability
- Queue scalability
- Compute scalability

---

# Observability Review

Verify:

- Logging
- Metrics
- Tracing
- Alerting

---

# Output

Generate:

- Architecture Findings
- Risks
- Tradeoffs
- Recommendations
- ADR Suggestions
