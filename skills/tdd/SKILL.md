# Test Driven Development (TDD)

Purpose

Ensure quality through tests before implementation.

---

# Workflow

1. Requirement
2. Acceptance Criteria
3. Test Cases
4. Failing Tests
5. Implementation
6. Refactor
7. Validation

---

# Test Design

Create:

- Happy Path Tests
- Edge Case Tests
- Negative Tests
- Security Tests
- Integration Tests

---

# Coverage Areas

Verify:

- Business Rules
- Error Handling
- Authorization
- Validation
- Data Integrity

---

# AI Solutions

Also create:

- Groundedness Tests
- Hallucination Tests
- Tool Calling Tests
- Prompt Injection Tests

---

# Deliverables

Generate:

/docs/tests/[feature-name]-test-plan.md

Include:

- Test Strategy
- Test Cases
- Risks
- Coverage Analysis

---

# Exit Criteria

Code is not complete until:

- Tests pass
- Critical paths are covered
- Acceptance criteria are validated
